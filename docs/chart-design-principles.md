# FencingAI Chart Design Principles

Charts should match the question users are asking. Horizontal progress bars are useful, but they should not become the only visual form.

## Use Progress Bars For Ratios

-晋级率
-淘汰赛完成率
-Bye 占比

These answer: "how much of the whole?"

## Use Bar Charts For Comparisons

-俱乐部参赛人数
-俱乐部前八数
-头部选手排名

These answer: "who is stronger or larger?"

## Use Path Flow For Elimination Journey

-冠军路径
-关键晋级过程

These answer: "how did this player advance?"

## Use Delta Charts For Rank Movement

-小组排名到最终排名的上升或下滑
-后程发力和掉队选手

These answer: "who changed position?"

## Product Rule

Text should explain the chart, not replace it. The first screen of a detail page should show a few high-signal visual conclusions, then let users inspect match details below.
