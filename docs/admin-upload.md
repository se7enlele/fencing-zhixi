# 本地解析后台使用说明

## 启动

在项目目录运行：

```bash
node server.mjs
```

看到以下输出后，用浏览器打开：

```text
Fencing data admin is running at http://127.0.0.1:5177
```

访问地址：

```text
http://127.0.0.1:5177
```

移动端查看页：

```text
http://127.0.0.1:5177/viewer
```

## 上传解析

页面支持上传：

- `.json`
- `.har`
- `.txt`

文件内容必须是合法 JSON 文本。系统只解析本地上传内容，不读取 Cookie，不读取鉴权信息，不向外部网站发起请求。

## 输出内容

后台会展示：

- 来源 URL 中解析出的 `sportId` 和 `eventCode`
- 候选数组路径
- 排名候选样本
- 对阵候选样本
- 字段出现频次
- 本次分析报告保存路径

报告会保存到：

```text
data/analysis/
```

## 验证命令

可用内置样本验证页面和 API：

```bash
node tools/smoke-test.mjs
```

## 当前 H5 信息路径

移动端页面按以下层级组织：

1. 地区/关键词筛选比赛
2. 进入比赛详情查看项目列表
3. 进入单项目查看分析

当前支持的查看内容：

- 比赛列表
- 比赛详情
- 项目概要
- 小组赛排名
- 小组赛比分样本
- 淘汰赛对阵
- 俱乐部分布
- 淘汰赛表现排行
